Part Five: Refine Customize USWDS settings

In the final part, we'll bring it all home and customize USWDS to better fit with the original project.

Stop the watch script.

Go the terminal window where the watch script is running and enter control-c to stop the script.

Activate theme settings.

We're going to add some custom USWDS settings to our project. To do this, we're going to add them to our custom settings file: _theme/_uswds-theme.scss.

In _theme/_uswds-theme.scss, uncomment lines 11-14

description _theme/_uswds-theme.scss

//  @use "uswds-core" with (
//    $setting: value,
//    $setting: value
//  );
@use "uswds-core" with (
  $setting: value,
  $setting: value
);

Update USWDS settings.

We can use USWDS settings to customize our installation and our compile settings.

While there's a comprehensive list of settings on the USWDS website at designsystem.digital.gov/documentation/settings/, we’ll just modifiy our installation a little. We’ll use Public Sans as our sans-serif font, give our banner a dark gray background color, and make it expand the full width of the page.

description _theme/_uswds-theme.scss

@use "uswds-core" with (
  $setting: value,
  $setting: value
  $theme-font-type-sans: "public-sans",
  $theme-banner-background-color: "ink",
  $theme-banner-max-width: none

);

Let's compile again to see the changes.

npx gulp watch

Remove unnecessary styles.

Now we've added the Banner and Identifier, but if we look at the size of our compiled CSS, it's quite large — over 500 KB! We can greatly reduce the size by using USWDS packages. You can learn more about packages at designsystem.digital.gov/components/packages, but for now, let's open up the USWDS Sass entry point (_theme/styles.scss), remove the main uswds package, and add only the usa-banner package.

description _theme/styles.scss

@forward "uswds-theme";
@forward "uswds";
@forward "usa-banner";
@forward "uswds-theme-custom-styles";

After you recompile, we can see that the CSS size drops down to 15 KB — less than 3% of the original size. It compiles in less than a second, and will gzip down to 2 KB in a production environment.

And that's how you get started with USWDS code in a project!
