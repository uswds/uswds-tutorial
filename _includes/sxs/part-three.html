Part Three: Compose the assets Add USWDS CSS and JavaScript our templates

So, now that we've created the compiled CSS and JavaScript, we need to include them in our page templates.

Add USWDS styles to the site head.

Add the compiled CSS before the existing site CSS.

description _includes/site/head.html

<head>
  <meta charset="utf-8">
  ...
  <link rel="stylesheet" href="/assets/css/open-props.css">
  <link rel="stylesheet" href="/assets/uswds/css/styles.css">
  <link rel="stylesheet" href="/assets/css/styles.css"> 
  ...

Add USWDS initializer to the site head.

Add the USWDS initialization script at the end of the page head. This script prevent flashes of unstyled content while USWDS is loading.

description _includes/site/head.html

<head>
  <meta charset="utf-8">
  ...
  <link rel="stylesheet" href="/assets/css/open-props.css">
  <link rel="stylesheet" href="/assets/uswds/css/styles.css">
  <link rel="stylesheet" href="/assets/css/styles.css">
  <script src="/assets/uswds/js/uswds-init.min.js"></script>

Add USWDS JavaScript to the end of the page body.

Finally, add the USWDS JavaScript to the end of the page body.

description ./_layouts/default.html

...

<body class="{{ page.body }}">
  {{ content }}
  <script src="/assets/uswds/js/uswds.min.js"></script>
</body>

Note: You've added USWDS styles and JavaScript to your project, but you shouldn't see any changes to the existing site. It still looks the same! We've designed USWDS for incremental adoption on existing sites. In most cases, you won't see the effects of USWDS code until you intentionally add USWDS components.

At this point, you've added USWDS design system code to your project successfully! Now we can add some components.
