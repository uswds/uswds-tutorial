So, now that we've created the compiled CSS and JavaScript, we need to include them in our page templates.

Add USWDS styles to the site head.

Add a link to the the compiled USWDS CSS before the existing site CSS.

description _includes/site/head.html

<head>
  <meta charset="utf-8">
  ...
  <link rel="stylesheet" href="/assets/css/open-props.css">
  <link rel="stylesheet" href="/assets/uswds/css/styles.css">
  <link rel="stylesheet" href="/assets/css/styles.css"> 
  ...

Add USWDS initializer to the site head.

Add the USWDS initialization script at the end of the page head. This script prevent flashes of unstyled content while USWDS is loading.

description _includes/site/head.html

<head>
  <meta charset="utf-8">
  ...
  <link rel="stylesheet" href="/assets/css/open-props.css">
  <link rel="stylesheet" href="/assets/uswds/css/styles.css">
  <link rel="stylesheet" href="/assets/css/styles.css">
  <script src="/assets/uswds/js/uswds-init.min.js"></script>

We've added USWDS styles and JavaScript to our project, but we don't see any changes to our site. This is because we've designed USWDS to keep its styles to itself. In most cases, you won't see the effects of adding USWDS code until you intentionally add a USWDS component.
